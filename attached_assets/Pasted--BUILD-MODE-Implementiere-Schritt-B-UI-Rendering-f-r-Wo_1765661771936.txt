[BUILD MODE]

Implementiere Schritt B (UI + Rendering) für “Workflow Connections” im LayoutEditor.

Ziel:
- Connect Mode Toggle im Header (neben Reset).
- In Connect Mode:
  - Räume sind NICHT draggable.
  - User kann von Raum A “ziehen” und auf Raum B loslassen → erstellt gerichtete Connection A→B.
  - ESC bricht den Verbindungsmodus während eines Ziehens ab.
- Rendern:
  - Lege ein SVG Overlay über die Canvas-Fläche (absolute inset-0).
  - Rendere jede Connection als cubic-bezier <path> mit Pfeilspitze.
  - Start/End: Seitenmitte zu Seitenmitte (je nach Richtung).
  - Linie ist gebogen (modern), strokeLinecap=round, strokeLinejoin=round.
  - Pfeile sollen “hübsch” wirken: keine harten Knicke, keine diagonalen geraden Striche.
- Während des Ziehens:
  - Preview-Connection folgt der Maus (auch als gebogener Pfeil, aber ohne Persistenz).
  - Wenn Maus über einem Room ist, highlight Target-Room.

Daten:
- Nutze react-query:
  - useQuery(["workflow-connections", practiceId]) → api.workflowConnections.list(practiceId)
  - create/update/delete Mutations mit invalidateQueries danach
- Minimal invasiv: LayoutEditor.tsx erweitern, keine großen Rewrites.

Wichtig zur Bezier-Geometrie:
- Berechne roomCenter = (x+width/2, y+height/2).
- Entscheide Horizontal vs Vertical dominant:
  - Wenn |dx| >= |dy|:
    - start = (from.x + (dx>0 ? from.width : 0), from.y + from.height/2)
    - end   = (to.x   + (dx>0 ? 0 : to.width),   to.y   + to.height/2)
    - control points: 
      c1 = (start.x + dx*0.5, start.y)
      c2 = (end.x   - dx*0.5, end.y)
  - Sonst:
    - start = (from.x + from.width/2, from.y + (dy>0 ? from.height : 0))
    - end   = (to.x   + to.width/2,   to.y   + (dy>0 ? 0 : to.height))
    - control points:
      c1 = (start.x, start.y + dy*0.5)
      c2 = (end.x,   end.y   - dy*0.5)
- Path: `M start.x start.y C c1.x c1.y c2.x c2.y end.x end.y`

Usability:
- Füge rechts oder links eine kleine Liste “Connections” ein:
  - zeigt fromRoomName → toRoomName
  - Delete Button pro Connection
- Kein Performance-Overkill: Connection Rendering soll ohne unnötige API calls laufen.

Output:
- Kurze Erklärung
- Betroffene Dateien
- Wie testen (Schritte im UI)
