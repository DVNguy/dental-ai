GO. Implementiere Milestone 1 Subphase A+B (Server Auth + Tenant-Scoping), minimal-invasiv.

A) Session + Auth
1) Aktiviere express-session mit connect-pg-simple als Store (Postgres).
   - Cookie: httpOnly=true, sameSite="lax", secure=true wenn NODE_ENV=production.
2) Implementiere server/auth.ts (oder server/routes/auth.ts) mit Routen:
   - POST /api/auth/register {username,password}
     * zod validation
     * bcrypt hash
     * user erstellen
     * optional: demo practice automatisch anlegen und session.practiceId setzen (wenn du das brauchst)
   - POST /api/auth/login {username,password} -> session setzen
   - POST /api/auth/logout -> session zerstören
   - GET /api/auth/me -> { userId, username, practiceId }
3) Keine Klartext-Passwörter. Keine Logs mit Passwort.

B) Tenant-Scoping
1) Baue Middleware:
   - requireAuth(req,res,next)
   - requirePracticeAccess(practiceId) (prüft, ob die Practice dem User gehört)
   - requireRoomAccess(roomId), requireStaffAccess(staffId), requireWorkflowAccess(workflowId), requireConnectionAccess(connectionId), requireStepAccess(stepId)
     * jeweils: DB lookup + join zu practices + ownership check
2) Wende requireAuth global auf alle /api/* an, außer /api/auth/*.
3) Wende Access-Middleware auf jede Route an, die practiceId oder entityId nutzt:
   - /api/practices/:id* -> requirePracticeAccess
   - /api/practices/:id/rooms|staff|workflows|workflow-connections|simulations -> requirePracticeAccess
   - /api/rooms/:id -> requireRoomAccess
   - /api/staff/:id -> requireStaffAccess
   - /api/workflows/:id -> requireWorkflowAccess
   - /api/workflow-connections/:id -> requireConnectionAccess
   - /api/workflow-steps/:id -> requireStepAccess
   - /api/ai/* und /api/v1/rag/* -> requireAuth + practice check, wo practiceId im Body steht
4) Debug endpoints: schalte /api/debug/status (falls vorhanden) in production ab (403).
